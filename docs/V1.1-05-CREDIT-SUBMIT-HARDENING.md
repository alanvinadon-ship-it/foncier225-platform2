# V1.1-05 Credit Submit Hardening

## Scope

This mission hardens the citizen-side credit journey before opening the future bank work queue.

Covered here:

- stronger dossier readiness messaging
- explicit missing document feedback
- submission confirmation
- post-submission read-only lock
- simple dossier timeline

Not covered here:

- bank portal
- scoring
- offers
- final decision
- final attestation workflow

## Submission rules

- A credit file can be submitted only from `DRAFT` or `DOCS_PENDING`
- All required document types must be present before submission
- A `SUBMITTED` file cannot be submitted again
- A `SUBMITTED` file cannot accept new citizen uploads

## Citizen UX behavior

- `DRAFT`: invite the user to start uploading documents
- `DOCS_PENDING`: highlight missing required documents
- `SUBMITTED`: show a persistent success state and lock mutations

## Upload behavior

- Allowed formats remain PDF, JPG, JPEG and PNG
- A single document is kept per credit document type in this citizen flow
- Uploading a new file for the same type replaces the previous file metadata

## Future extension points

- replace base64 upload with a presigned flow
- feed the bank queue from `SUBMITTED` files
- add richer audit-backed timeline events
- connect the post-review bank states without changing citizen locking rules
