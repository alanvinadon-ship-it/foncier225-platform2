## Upload strategy

No reusable browser upload flow was present in the repository UI.

For this mission, the citizen flow uses a minimal secure path:

1. the browser reads the selected file,
2. it sends the payload as base64 to `credit.addCreditDocument`,
3. the server validates owner, document type, MIME type and size,
4. the server uploads the binary through `storagePut`,
5. the uploaded file is attached to `credit_documents`.

## Current limits

- Allowed formats: PDF, JPG, JPEG, PNG
- Max size: 10 MB
- Uploads are stored through the existing storage proxy helper
- The flow does not yet implement presign/complete split uploads
- Documents are attached to `credit_documents`; deeper synchronization with the generic `documents` module can be expanded later if needed

## Future extensions

- Replace base64 upload with presigned direct upload when storage flow is industrialized
- Add richer submit UX with confirmation and post-submit state transitions
- Expose bank review surfaces in a later mission without changing citizen routes
