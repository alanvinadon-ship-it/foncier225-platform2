# V1.2-02 Request Docs, Offer and Decision

## Scope

This mission extends the bank workflow after `UNDER_REVIEW`:

- request additional documents
- emit an offer
- let the citizen accept or reject the offer
- let the bank issue a final decision after acceptance

Out of scope:

- final attestation PDF + QR
- final documentary issuance worker
- full consent model
- scoring engine

## Workflow covered

- `UNDER_REVIEW` + `REQUEST_DOCS` -> `DOCS_PENDING`
- `UNDER_REVIEW` + `MAKE_OFFER` -> `OFFERED`
- `OFFERED` + `ACCEPT_OFFER` -> `ACCEPTED`
- `OFFERED` + `REJECT_OFFER` -> `CLOSED`
- `ACCEPTED` + `APPROVE` -> `APPROVED`
- `ACCEPTED` + `REJECT` -> `REJECTED`

## Offer strategy

The schema already supports multiple offers historically.

Chosen behavior:

- keep offer history
- expire previous `pending` offers when a new offer is created
- treat the latest offer as the active offer shown in UI

This keeps the model simple while preserving traceability.

## Citizen visibility

The citizen detail page now exposes:

- open and past complement requests
- latest offer
- accept / reject actions for the owner only
- final decision state when available

## Future extensions

- richer request fulfillment tracking
- explicit notifications
- attestation issuance after `APPROVED`
- tighter consent-based visibility rules if the model evolves
