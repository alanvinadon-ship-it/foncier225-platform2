# V1.2-01 Bank Work Queue

## Scope

This mission opens the first bank-side review surface for credit habitat files.

Delivered here:

- dedicated bank layout
- bank work queue for `SUBMITTED` and `UNDER_REVIEW`
- bank detail page
- controlled read access
- `REVIEW` transition from `SUBMITTED` to `UNDER_REVIEW`
- bank audit events for view and review

Not delivered here:

- request docs workflow
- document validation/rejection workflow
- offer workflow
- final decision workflow
- final attestation workflow

## Access assumption

At this stage, no fine-grained bank assignment model is visible in the repository.

Chosen behavior:

- any authenticated user with role `bank` can read `SUBMITTED` and `UNDER_REVIEW` files
- this is intentionally narrow and can later be replaced with bank assignment rules

## Visible data

The bank detail view stays minimal:

- dossier summary
- parcel reference when available
- applicant name/email when already present in the current model
- document list
- checklist state

This keeps the portal useful without pre-implementing the future consent model.

## Workflow rule

- only `SUBMITTED` files can transition to `UNDER_REVIEW`
- invalid transitions are rejected and audited
