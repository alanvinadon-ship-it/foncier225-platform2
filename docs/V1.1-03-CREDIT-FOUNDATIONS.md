# V1.1-03 Credit Foundations

## Scope

This module prepares the server-side foundations for housing credit files without opening public access or refactoring the existing parcel, verify, or citizen areas.

## Statuses

- `DRAFT`
- `DOCS_PENDING`
- `SUBMITTED`
- `UNDER_REVIEW`
- `OFFERED`
- `ACCEPTED`
- `APPROVED`
- `REJECTED`
- `CLOSED`

## Supported transitions

- `DRAFT + ADD_DOC -> DOCS_PENDING`
- `DRAFT + SUBMIT -> SUBMITTED`
- `DOCS_PENDING + UPLOAD_COMPLETE -> DOCS_PENDING`
- `DOCS_PENDING + SUBMIT -> SUBMITTED`
- `SUBMITTED + REVIEW -> UNDER_REVIEW`
- `UNDER_REVIEW + REQUEST_DOCS -> DOCS_PENDING`
- `UNDER_REVIEW + MAKE_OFFER -> OFFERED`
- `OFFERED + ACCEPT_OFFER -> ACCEPTED`
- `OFFERED + REJECT_OFFER -> CLOSED`
- `ACCEPTED + APPROVE -> APPROVED`
- `ACCEPTED + REJECT -> REJECTED`

## Extension points

- Reuse `documents` for physical file ownership and storage integration.
- Use `credit_documents` as the business layer for required credit pieces.
- Keep bank queue, consent, scoring, and final PDF/QR attestation for later missions.
- Keep all access private and owner-scoped until dedicated bank/admin routes are introduced.
